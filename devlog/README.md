# Devlog

This folder keeps planning-with-files artifacts as dated work logs.

## Layout

- Use one folder per work day named with an ISO date: `YYYY-MM-DD`
- Keep the three planning files together inside that folder:
  - `task_plan.md`
  - `findings.md`
  - `progress.md`

## Example

`devlog/2026-03-14/task_plan.md`

## Notes

- Continue the same day's work in the existing dated folder.
- Create a new dated folder when a new day starts.
- Leave older entries in place so past implementation history stays easy to review.
